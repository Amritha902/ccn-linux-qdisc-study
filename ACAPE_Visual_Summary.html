SECTION 01

Four Novel Contributions

ACAPE fills a confirmed gap: no prior work tunes fq_codel's four parameters at runtime using gradient prediction + eBPF telemetry on an unmodified kernel.

C1 — GRADIENT ESTIMATOR

Multi-Signal State Vector

S(t) = [∇dr, ∇bl, ∇rtt] — linear regression over W=10 samples. No prior AQM fuses all three gradients.

C2 — PREDICTIVE CONTROL

Regime Prediction

Acts on PREDICTED next state when trajectory is WORSENING. All prior adaptive AQM (incl. Adaptive RED) is purely reactive.

C3 — WORKLOAD PROFILES

eBPF Flow Classification

Selects MICE / MIXED / ELEPHANT profiles from live BPF map elephant_ratio. First system to use eBPF data for fq_codel profile selection.

C4 — THREE TIMESCALES

Stock Linux Deployment

T1 (eBPF ~ns) → T2 (gradient 500ms) → T3 (AIMD ~5s). Grounded in Borkar's stochastic approximation theory. Zero kernel modification.

SECTION 03

Complete Algorithm Reference

Gradient Estimation (C1)

EQ 1 — LINEAR REGRESSION SLOPE
∇f(t) = Σᵢ(tᵢ − t̄)(f(tᵢ) − f̄) / Σᵢ(tᵢ − t̄)²
f ∈ {drop_rate, backlog, rtt_proxy} · window W=10 samples · computed each T2=500ms tick
EQ 2 — COMPOSITE GRADIENT
G(t) = 0.6 × (∇dr / DR_HEAVY) + 0.4 × (∇bl / BL_HEAVY) where DR_HEAVY=30 drops/sec, BL_HEAVY=300 pkts
EQ 3 — TRAJECTORY
trajectory = WORSENING if G(t) > 0.5 = RECOVERING if G(t) < −0.5 = STABLE otherwise

Regime Classification & Prediction (C2)

EQ 4 — CURRENT REGIME
regime = HEAVY if dr > 30 OR bl > 300 = MODERATE if dr > 10 OR bl > 100 = LIGHT if dr > 1 OR bl > 20 = NORMAL otherwise
EQ 5 — PREDICTED NEXT REGIME (NOVEL)
predicted = REGIMES[idx+1] if WORSENING and idx < 3 = REGIMES[idx-1] if RECOVERING and idx > 0 = regime otherwise REGIMES = [NORMAL, LIGHT, MODERATE, HEAVY]
EQ 6 — EFFECTIVE REGIME FOR AIMD
effective = predicted if trajectory = WORSENING → [PREDICTIVE] tag = regime otherwise → [REACTIVE] tag
This is the core novelty: ACAPE acts on the future state, not the current one

Workload Profile Selection (C3)

EQ 7 — ELEPHANT RATIO (FROM eBPF MAPS)
elephant_ratio = |{flows: bytes > 10MB, age < 2s}| / active_flows
EQ 8 — WORKLOAD PROFILE
workload = MICE if ratio < 0.2 → target=2ms, quantum=300B = ELEPHANT if ratio > 0.6 → target=10ms, quantum=3000B = MIXED otherwise → target=5ms, quantum=1514B

AIMD Parameter Adjustment

EQ 9 — AIMD RULES (β=0.9, α=0.5ms)
HEAVY: target × 0.9 | interval × 0.9 | limit × 0.9 MODERATE: target − 0.2 | limit − 32 LIGHT: target + 0.5 | interval + 5ms | limit + 64 NORMAL+RECOVERING: target + 0.2 | limit + 16 quantum ← WORKLOAD_PROFILES[workload] ← (C3)
β=0.9 from Adaptive RED (Floyd 2001). α=0.5ms selected empirically.
EQ 10 — PARAMETER BOUNDS
target ∈ [1ms, 20ms] interval ≥ target × 10 (CoDel constraint) interval ∈ [50ms, 300ms] limit ∈ [256, 4096] packets quantum ∈ {300, 1514, 3000} bytes (from workload profile)

eBPF RTT Proxy

EQ 11 — IN-KERNEL EWMA (per-packet, TC hook)
gap_ns = gap_ns==0 ? Δt : (gap_ns×7 + Δt) >> 3 α = 0.125 (same as TCP RTT estimator, RFC 6298) where Δt = now_ns − last_seen_ns
EQ 12 — AGGREGATE RTT PROXY (userspace, T2)
rtt_proxy_ms = mean(gap_ns over active flows, age < 2s) / 1e6

eBPF Pipeline

KEY INSIGHT

BPF maps are kernel-global objects. Even though tc_monitor.o is attached inside ns1, the prog ID and map IDs are visible from the host via bpftool. This is why ACAPE does not need BCC, nsenter, or pyroute2 for map reads — standard bpftool map dump id N --json from the host returns all flow_map entries.

SECTION 06

Complete Implementation Steps

  1. Install dependencies
    sudo apt install -y clang llvm libbpf-dev iperf3 iproute2 linux-tools-$(uname -r) bpftool
    pip3 install matplotlib numpy --break-system-packages
  2. Build eBPF object
    cd ~/ccn-linux-qdisc-study/ebpf && make clean && make
    Must print: ✅ Built: tc_monitor.o
  3. Setup namespace testbed
    sudo bash ~/ccn-linux-qdisc-study/scripts/setup_ns.sh
    OR run manual commands (see README). Must show: 0% packet loss
  4. Start iperf3 server (Terminal 1)
    sudo ip netns exec ns1 iperf3 -s
  5. Start traffic (Terminal 2)
    sudo ip netns exec ns2 iperf3 -c 10.0.0.1 -P 8 -t 120 -i 1 --logfile ~/ccn-linux-qdisc-study/logs/iperf_$(date +%H%M%S).log
  6. Run ACAPE controller (Terminal 3) — wait 5s after traffic starts
    cd ~/ccn-linux-qdisc-study/scripts
    sudo python3 acape_complete.py --ns ns1 --iface veth1 --logdir ../logs
  7. Wait for experiment to complete, then Ctrl+C, then plot
    python3 plot_acape.py --logdir ../logs
  8. Commit and push
    cd ~/ccn-linux-qdisc-study && git add . && git commit -m "ACAPE: results" && git push
VERIFY eBPF IS ACTIVE

At startup you must see: [eBPF] ✅ All 3 layers active and the flows column showing non-zero numbers. If flows=0, check: bpftool version and sudo ip netns exec ns1 tc filter show dev veth1 egress
